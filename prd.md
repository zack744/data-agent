## 1. 项目概述

| 项目属性 | 内容 |
| -------- | ---- |
| **项目名称** | 智能选题决策与分析报告生成平台（内部代号：Topic Agent） |
| **项目愿景** | 为内容创作者打造一个由多 AI Agent 协同工作的自主分析平台，提供端到端的智能服务：从接收一个想法，到自动抓取数据、深度分析，最终生成一份带有预测和决策建议的专业分析报告。 |
| **核心价值** | 自动化与智能化。将创作者从繁重的数据收集和基础分析中解放出来，直接获得类似专业数据分析师提供的深度洞察。 |
| **项目定位** | 一个以 AI Agent 为“大脑”，以数据爬虫为“手脚”的 MVP 系统。前端界面仅是任务的发起点和报告的展示窗。 |

## 2. 核心变革：AI Agent 驱动的范式转移

| 特性 | 传统选题工具（参考常见毕业设计选题系统） | 本系统（AI Agent 核心） |
| ---- | --------------------------------------- | ------------------------ |
| **分析方式** | 人工查看仪表盘和图表，自行解读。 | AI Agent 自动解读数据，发现人眼不易察觉的模式、关联性和异常点。 |
| **输出成果** | 静态的图表和数字列表。 | 动态生成的、结构化的、图文并茂的 Markdown/PDF 格式分析报告。 |
| **交互模式** | 人操作工具，点击筛选。 | 人与智能体对话。用户可用自然语言提出复杂需求。 |
| **核心能力** | 数据可视化与增删改查。 | 数据推理、洞察生成、自然语言撰写、预测判断。 |

## 3. 系统架构（以 AI Agent 为核心）

系统业务流程由多个专职 AI Agent 协同完成：

### 3.1 核心 AI Agent 角色定义

#### ① 任务规划与调度 Agent（大脑）
- **职责**：理解用户的自然语言指令，拆解成可执行的任务链。
- **示例**：爬取 B 站/小红书数据 → 时序分析 → 生成对比报告。
- **技术**：LLM Function Calling（如 GPT-4、DeepSeek）。

#### ② 数据采集与处理 Agent（手脚）
- **职责**：接收爬虫任务，执行脚本，清洗数据，入库。
- **技术**：Python 爬虫（Playwright/Scrapy）+ LLM 辅助解析不规则文本。

#### ③ 数据分析与洞察 Agent（核心分析师）
- **职责**：
  - 量化分析：计算“三类七字段”指标。
  - 趋势预测：轻型时序模型预测热度走势。
  - 深度洞察：竞争格局、内容模板、风险识别。
- **技术**：LLM + Scikit-learn（轻量化 ML）。

#### ④ 报告撰写与呈现 Agent（专业撰稿人）
- **职责**：将结构化洞察按模板生成图文报告（Markdown 优先）。
- **技术**：LLM 内容生成 + 模板引擎（Jinja2/Markdown）。

## 4. 功能需求（MVP 版本）

### 4.1 用户角色与核心流程

| 用户角色 | 核心目标 |
| -------- | -------- |
| 内容创作者 | 快速获得数据支撑的选题决策建议，并理解该如何操作。 |

**MVP 核心流程**：
1. 输入：Web 界面输入关键词（如“Citywalk”）。
2. 触发：点击“生成分析报告”。
3. 处理：后台调度 AI Agent 链工作。
4. 输出：界面显示进度 → 呈现完整分析报告 → 支持下载。

### 4.2 MVP 功能清单

| 功能模块 | 功能点 | 说明 | 优先级 |
| -------- | ------ | ---- | ------ |
| **任务输入** | 关键词输入 | 支持单关键词输入 | P0 |
|  | 平台选择 | MVP 优先支持 B 站 | P0 |
| **智能分析** | 自动数据采集 | Agent 爬取并清洗“三类七字段”数据 | P0 |
|  | 自动数据分析与洞察 | 计算指标、趋势预测、生成结论 | P0 |
|  | 自动报告生成 | 输出 Markdown 格式报告 | P0 |
| **结果呈现** | 报告可视化展示 | 前端渲染 Markdown（含图表、表格） | P0 |
|  | 报告导出 | 支持 Markdown/PDF 下载 | P1 |

## 5. 核心数据指标（AI Agent 分析依据）

| 类别 | 字段 | 分析目标 |
| ---- | ---- | -------- |
| **需求侧** | 播放量/阅读量 | 衡量市场天花板 |
|  | 点赞率（点赞/播放） | 衡量内容质量与用户满意度 |
| **供给侧** | 同题作品数 | 竞争激烈程度（红海/蓝海） |
|  | 头部创作者占比 | 竞争壁垒高度 |
| **时间侧** | 发布时间点 | 流量窗口规律 |
|  | 24 小时播放增速 | 判断是否正兴起 |
| **文本侧** | 关键词出现频次 | 发现爆款内容结构与模板 |

---

## 6. 非功能性需求

- **性能**：5 分钟内完成报告生成（大部分场景）。
- **可靠性**：采集 Agent 具备容错机制，应对目标网站结构变动。
- **可用性**：极致简化交互，用户仅输入关键词+点击按钮。

---

## 7. 技术栈考量

| 层级 | 技术选型 |
| ---- | -------- |
| **后端核心** | Python + FastAPI（API & Agent 通信） |
| **AI 模型层** | LLM：DeepSeek-V2 / GPT-4o-mini；轻型 ML：Scikit-learn |
| **数据存储** | PostgreSQL（结构化数据） |
| **前端** | Vue.js/React + Markdown 渲染器，或 Streamlit 快速原型 |
| **文档规范** | Markdown（统一书写规范） |

---

## 8. 后续演进方向

1. **平台扩展**：接入小红书、抖音等更多平台。
2. **分析深化**：引入复杂智能匹配与推荐算法。
3. **交互升级**：完整自然语言对话，支持追问。
4. **功能丰富**：选题监控预警、竞品跟踪等。
