# 项目简报 (Project Brief)

## 项目概述
**项目名称**：智能选题决策与分析报告生成平台
**代号**：Topic Agent
**类型**：AI驱动的数据分析工具
**阶段**：MVP开发完成，待测试和部署

## 项目目标

### 主要目标
1. **自动化数据分析**：减少内容创作者的数据收集和分析工作量
2. **AI增强决策**：利用GPT模型生成专业的分析洞察和标题建议
3. **快速报告生成**：5分钟内生成完整的分析报告
4. **平台无关架构**：通过适配器模式支持多平台数据源

### 成功标准
- [ ] 能够成功采集和解析B站热搜数据
- [ ] 适配器正确转换数据格式，保留原始信息
- [ ] AI模型能够生成有意义的标题建议
- [ ] 生成的报告格式规范，内容结构清晰
- [ ] API接口稳定可用，响应时间合理
- [ ] 整体系统架构清晰，便于扩展和维护

## 项目范围

### 包含范围
✅ **数据采集模块**
- B站热搜数据爬虫
- 数据清洗和结构化存储
- 按日期组织的JSON文件存储

✅ **AI分析模块**
- LangChain Agent框架
- 基础统计分析（数量、浏览量、点赞率）
- OpenAI GPT模型集成
- 标题生成功能

✅ **报告生成模块**
- Jinja2模板引擎
- Markdown格式输出
- 结构化报告模板

✅ **API服务模块**
- FastAPI Web框架
- `/health`健康检查接口
- `/report`分析报告接口
- RESTful API设计

✅ **配置管理**
- 环境变量配置
- API密钥管理
- 多模型支持（OpenAI兼容）

### 排除范围
❌ **前端界面**：暂无Web UI，仅提供API接口
❌ **用户管理**：无用户认证和权限控制
❌ **数据库**：使用JSON文件而非数据库存储
❌ **实时采集**：需要手动触发数据采集
❌ **多平台**：暂只支持B站，其他平台待开发
❌ **高级分析**：暂无复杂的机器学习模型

## 技术约束

### 技术栈约束
- **语言**：Python 3.10+
- **AI框架**：LangChain (不能更换)
- **Web框架**：FastAPI (已选定)
- **数据格式**：JSON (简化存储)

### 性能约束
- **响应时间**：报告生成 < 5分钟
- **并发支持**：单用户访问（无并发需求）
- **数据规模**：适合中小规模数据集（< 10000条记录）
- **资源消耗**：AI API调用频率受限，需控制成本

### 兼容性约束
- **API兼容**：支持OpenAI兼容的API端点
- **操作系统**：跨平台（Windows/Linux/Mac）
- **Python版本**：3.10+（利用新特性）

## 项目背景

### 市场需求
内容创作者在选题时往往依赖经验和直觉，缺乏数据支撑。传统的数据分析工具门槛高、流程复杂，中小创作者难以使用。

### 解决方案
通过AI Agent自动化整个数据分析流程，让任何人都能快速获得专业级的分析报告。

### 技术可行性
- LangChain生态成熟，集成简单
- OpenAI API稳定，成本可控
- B站API可获取基础数据
- Python生态丰富，开发效率高
